Literature Survey

This literature review explores the integration of Natural Language Processing (NLP) and Machine Learning (ML) into software development, specifically focusing on automating tasks such as user story generation, use case diagram generation, class diagram generation, predicting design patterns, and generating skeleton code. Güneş and Aydemir’s innovative pipeline leverages NLP to generate goal models from user stories, demonstrating that these models closely resemble those created by humans. This approach not only enhances developers’ ability to understand and structure backlog items but also signifies a promising step towards automating traditionally human-dependent tasks, thereby improving efficiency in the software development lifecycle. However, the automated generation of user stories remains underexplored, indicating a need for further research in this area.

In addition, the work by Narawita and Vidanage presents a system combining NLP and ML to create use case diagrams, although it currently lacks the capability to identify certain relationship types such as include, extend, and generalize relationships. Future improvements may address these limitations by incorporating more data and domain-specific adjustments. Nasiri's study uses "Stanford Core NLP" to generate class diagrams in XML format from user story specifications, proposing enhancements such as incorporating cardinalities and leveraging AI for better rule formulation. The realm of UML-based code generation has seen methodologies focusing on generating Java files that detail class attributes and relationships, while Tsantalis’s approach to design pattern recognition highlights the limitations of similarity scoring algorithms, which compare nodes rather than entire graphs. These studies collectively reveal both the potential and the challenges of integrating NLP and ML into software development practices, highlighting the ongoing need for advancements and refinements in these technologies.

Research Gap

User Story Generation

Limited research explores automated extraction of goal models from user stories using NLP, highlighting a need to improve requirements understanding and traceability. Quality issues in manually crafted user stories underscore the need for automated methods to ensure clarity and reduce ambiguity.

Use Case Diagram Generation

Current studies on automated UML diagram generation using NLP fail to accurately identify "generalization" relationships, crucial for system hierarchy. There is also a gap in recognizing complex relationships like "include" and "extend," requiring advanced NLP and ML techniques.

Class Diagram Generation

Research on UML class diagram generation needs to address complex scenarios such as recursive relationships and inheritance hierarchies. There is also a need for improved scalability and adaptability to different languages and complex textual structures in existing studies.

Skeleton Code Generation and Design Pattern Prediction

There is a lack of methods for generating skeleton code from identified design patterns, despite techniques for checking UML design patterns in Java. Existing research does not define mapping rules using class diagrams for design pattern identification, presenting an opportunity for novel approaches.

Research Problem & Solution

 

1. User Story Generation

Manual creation of user stories in Agile software development faces challenges of inconsistency, ambiguity, and time constraints, necessitating automated approaches to enhance quality, coherence, and alignment with Agile methodologies.

 

2. Use Case Diagram Generation

Automating the generation of use case diagrams encounters difficulties in accurately identifying relationships between actors and use cases, especially complex ones, requiring advanced NLP and ML techniques to improve contextual understanding and representation of system interactions.

 

3. Class Diagram Generation

Manual creation of user stories in Agile software development faces challenges of inconsistency, ambiguity, and time constraints, necessitating automated approaches to enhance quality, coherence, and alignment with Agile methodologies.

 

4. Skeleton Code Generation and Design Pattern Prediction

Automating the identification of design patterns in UML class diagrams is challenging due to the complexity and context-specific nature of patterns, necessitating advanced techniques to improve efficiency, customization, and the adoption of best practices in software development.

Methodology

Figure 2. High Level Architecture of the system.

The proposed UML diagram generation and Design Pattern integration system consists of 4 main components. They are:

  1. Generating user stories from client scenario.
  2. Generating use case diagrams from the client scenario.
  3. Generating class diagrams from the client scenario.
  4. Generating source code according to the design pattern from the class diagram.

Fig 4 illustrates the overall system diagram of the proposed solution which was intended to provide a smart approach for stakeholders, researchers, and Coconut Development Officers (CDOs) to detect the coconut diseases and pest infestations that may affect the coconut industry. As shown in the diagram, the registered users of the system can capture or upload the images which are suspicious. The images are sent to the Amazon Web Services (AWS) backend server where the flask server is deployed. These images are processed in the flask server by the designed DCNN models for disease identification. If WCLWD is found, the symptom severity will be determined using CNN models. Simultaneously, if CCI is identified, the images will go through the Mask-R-CNN model to determine the progression level while the number of caterpillars are extracted using the YOLOv5 object detection algorithm. Images will be classified using the CNN models of Mg deficiency and LSD at the same time.

Once the system identifies that the leaves are infected, then the response will be captured by the crowdsourcing platform. The Google Map will be updated with the real-time locations (latitude and longitude) of the infected palms. In addition, the system will automatically send notifications to the farmers and other stakeholders who are at the risk of infection.
